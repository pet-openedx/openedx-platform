Local Development

Running LMS and CMS directly on macOS with Tutor-managed backing services

Architecture

All backing services run in Docker via Tutor. LMS and CMS run directly on the host inside a Python virtualenv so you can attach a debugger and iterate without rebuilding containers.

First-time setup

Run these once after cloning. Requires Homebrew, pyenv, Node 20, and Tutor installed.

1

Install system dependencies

C libraries needed to compile Python packages on macOS.

make macos-requirements
2

Create and activate Python virtualenv

pyenv install 3.11.11   # if not already installed
python -m venv ~/venvs/openedx
source ~/venvs/openedx/bin/activate
3

Install Python dependencies

make dev-requirements
4

Install Node dependencies

npm install
5

Start backing services

make run-services
6

Run database migrations

make migrate-lms
make migrate-cms
7

Create a superuser one-time

LMS_CFG="$HOME/Library/Application Support/tutor/env/apps/openedx/config/lms.env.yml" \
  python manage.py lms createsuperuser

Daily workflow

Step 1 — Start backing services

make run-services

Starts MySQL (3306), MongoDB (27017), Redis (6380) in Docker with ports exposed to host. Only needed once per machine restart.

Step 2 — Run the servers (separate terminals)

source ~/venvs/openedx/bin/activate
make run-lms   # terminal 1  →  http://localhost:8000
make run-cms   # terminal 2  →  http://localhost:8001

Step 3 — Build static assets (optional)

HTML pages (login, dashboard, etc.) require compiled assets. REST APIs work without them.

make build-assets    # one-time production-style build
# or during active JS development:
make watch-assets    # rebuilds on file changes
Note: Without compiled assets, HTML pages return 500. REST APIs and admin still work.

MFE development

Point your MFE's .env.development at http://localhost:8000. The LMS is already configured to allow CORS from localhost:2000.

LMS_BASE_URL=http://localhost:8000
LOGIN_URL=http://localhost:8000/login
LOGOUT_URL=http://localhost:8000/logout
