Health Check

Built-in heartbeat endpoint for verifying the status of LMS and its backing services

Endpoint

The LMS exposes a single heartbeat endpoint at /heartbeat, mounted by the openedx.core.djangoapps.heartbeat app. It is intended for load balancers and monitoring tools.

Normal mode

GET http://localhost:8000/heartbeat
  • MySQL (SQL ping)
  • MongoDB / modulestore

Extended mode slower

GET http://localhost:8000/heartbeat?extended
  • MySQL (SQL ping)
  • MongoDB / modulestore
  • Celery worker + Redis broker
Note: The extended check dispatches a real Celery task and waits up to HEARTBEAT_CELERY_TIMEOUT seconds (default 5 s) for a result. Don't use ?extended for frequent automated probes — use it for manual diagnosis.

What each check verifies

KeyModeServiceWhat it does
sql normal MySQL :3306 Executes SELECT 1 via Django's DB connection
modulestore normal MongoDB :27017 Calls modulestore().heartbeat() — pings all configured stores (Split + Mongo)
celery extended Celery worker + Redis :6380 Dispatches a no-op task via sample_task.apply_async() and polls for completion within the timeout

What is NOT checked

The Redis cache (read/write) is not included in the default checks. The functions check_cache_set and check_cache_get exist in default_checks.py but are not wired into HEARTBEAT_CHECKS. The extended Celery check does implicitly exercise Redis as a broker.

Response format

200 OK — all checks passed

{
  "sql":         { "status": true, "message": "OK" },
  "modulestore": { "status": true, "message": "OK" }
}

200 OK — extended mode, all passed

{
  "sql":         { "status": true, "message": "OK" },
  "modulestore": { "status": true, "message": "OK" },
  "celery":      { "status": true, "message": "{'time': '0.312'}" }
}

503 Service Unavailable — one or more checks failed

{
  "sql":         { "status": true,  "message": "OK" },
  "modulestore": { "status": false, "message": "MongoDB connection refused" }
}

A 503 is returned if any single check returns false. The response body identifies which service failed and why.

Configuration

SettingDefaultDescription
HEARTBEAT_CHECKS [check_modulestore, check_database] Checks run in normal mode. Add dotted-path strings to include custom checks.
HEARTBEAT_EXTENDED_CHECKS (check_celery,) Additional checks run only when ?extended is present.
HEARTBEAT_CELERY_TIMEOUT 5 (seconds) How long to wait for the Celery sample task before declaring failure.
HEARTBEAT_CELERY_ROUTING_KEY HIGH_PRIORITY_QUEUE Queue the sample task is sent to.
